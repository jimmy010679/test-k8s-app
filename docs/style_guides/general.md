# 一般程式碼風格指南 (General Code Style Guide)

本文件概述了適用於 **test-k8s-app** 專案中所有檔案的一般編碼標準，不論使用何種程式語言。

## 1. 行長度 (Line Length)
- **最大行長度：** 預設情況下，所有程式碼和文件的每一行應限制在最多 80 個字元以內。
- **例外情況：** 此規則普遍適用，除非特定語言的風格規則有明確覆蓋（例如：如果特定語言的 Linter 嚴格要求某些 Import 或產生的程式碼必須使用更長的行）。然而，在缺乏此類強制性例外的情況下，必須嚴格遵守 80 個字元的限制。

## 2. 尾隨空白 (Trailing Whitespace)
- **尾隨空白：** 任何檔案中均不允許出現尾隨空白 (Trailing Whitespace)。這適用於所有檔案，不論其使用的程式語言為何。
